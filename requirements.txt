# DisTraceAI shared pipeline requirements (BOTH backends).
#
# The inference library itself is NOT pinned here — vLLM is installed by the
# vllm branch of setup.sh, llama-cpp-python is built from source in the
# llama-cpp branch. Pinning vllm here would force-pull it into the llama-cpp
# env (slow + incompatible CUDA stack).
#
# transformers is also left unpinned: vLLM pulls v5; the llama-cpp branch
# resolves a recent v4. The check-worthiness detectors (mDeBERTa / XLM-R) work
# on both.

# retrieval
# NodeRAG is NOT installed from PyPI — it's a patched local clone installed by
# modules/noderag/install_noderag_local.sh (called from setup.sh). Listing it
# here would either fail to resolve or install an UNPATCHED copy that conflicts
# with the editable install, and a failed line aborts the rest of this file.
chromadb

numpy

scikit-learn

# narwhals >= 2.0. The llama-cpp env was hitting a transitively-pinned older

# moment sentence-transformers imported sklearn. Pinning explicitly here
# stops the resolver from picking a 1.x backslide.
